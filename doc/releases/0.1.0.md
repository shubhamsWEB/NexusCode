# Release Notes — v0.1.0

**Release date:** 2026-02-22

`v0.1.0` is the first public release of **NexusCode**, a self-hosted codebase intelligence server that keeps GitHub repositories continuously indexed and exposes them to AI agents through MCP and REST APIs.

This launch focuses on the core product loop: register a repo, ingest code through GitHub, search it with hybrid retrieval, and hand grounded context to coding agents and planning workflows.

---

## Highlights

- **End-to-end indexing pipeline** from GitHub webhook to PostgreSQL, including Tree-sitter parsing, recursive chunking, scope-aware enrichment, and embeddings generation.
- **Hybrid code retrieval** that combines semantic vector search, keyword matching, reciprocal rank fusion, and reranking for higher-quality results.
- **MCP integration for AI agents** with 5 launch tools: `search_codebase`, `get_symbol`, `find_callers`, `get_file_context`, and `get_agent_context`.
- **Planning Mode** via `POST /plan`, combining codebase retrieval with web research to generate grounded implementation plans.
- **Operational baseline** with Docker Compose, Redis-backed background indexing, a Streamlit admin dashboard, and Railway deployment support.

## What Ships In This Release

### Core platform

- FastAPI application with health, repository management, search, planning, and webhook endpoints.
- Docker Compose development stack for PostgreSQL with `pgvector`, Redis, API server, worker, and dashboard.
- Background indexing through RQ workers for full and incremental repository ingestion.

### GitHub ingestion

- GitHub repository registration and fetch flow.
- Push webhook receiver with HMAC-SHA256 verification.
- Merkle-based incremental indexing so only changed files are reprocessed after a push.

### Retrieval and code understanding

- Tree-sitter AST parsing across 12+ languages.
- Symbol extraction and scope-chain enrichment.
- Voyage-powered code embeddings.
- Hybrid retrieval pipeline with semantic search, `pg_trgm` keyword search, RRF merge, and cross-encoder reranking.

### Agent and developer interfaces

- MCP server exposed over SSE at `/mcp`.
- Planning endpoint at `POST /plan`.
- Streamlit dashboard for repo management, health checks, webhook setup, and planning workflows.
- OAuth 2.1 + PKCE support for MCP authentication.

## Intended For

This release is aimed at teams that want one shared, always-fresh code intelligence layer instead of having each agent or engineer repeatedly read repositories from scratch.

It is especially useful when you want to:

- keep multiple AI agents in sync with the latest state of a repo,
- answer structural code questions through search and symbol-aware retrieval,
- generate implementation plans grounded in both the current codebase and external references,
- run the full stack locally or deploy it with minimal infrastructure setup.

## First-Release Scope

`v0.1.0` establishes the foundation of NexusCode. It does **not** include several features added after the initial release, such as Ask Mode, workflows, cross-repo routing, scoped API keys, PDF generation, knowledge graph views, external MCP server bridging, or custom agent roles.

If you are reading newer docs in this repository, use this note as the source of truth for what was included in the first release.

## Getting Started

To try the initial release workflow:

1. Configure `DATABASE_URL`, `REDIS_URL`, `VOYAGE_API_KEY`, `ANTHROPIC_API_KEY`, `GITHUB_TOKEN`, `GITHUB_WEBHOOK_SECRET`, and `JWT_SECRET`.
2. Start PostgreSQL and Redis, then run the API server, indexing worker, and dashboard.
3. Register a repository with `POST /repos`.
4. Trigger indexing and connect an MCP client or call `POST /search` and `POST /plan`.

See [Getting Started](../getting-started.md), [Connecting GitHub](../connecting-github.md), and [MCP Access](../mcp-access.md) for the current setup guides.
