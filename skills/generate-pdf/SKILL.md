# generate_pdf — PDF Export Tool

Convert any markdown document into a professional, downloadable PDF in one tool call.

## When to Use

- After writing a long-form document (RCA, report, spec) that should be shared externally
- When a workflow step produces a markdown artifact that stakeholders need as a file
- When explicitly asked to "generate a PDF", "export as PDF", or "create a downloadable report"

## Tool: `generate_pdf`

Available to agents with the `supervisor` role (or any role with `generate_pdf` in its `default_tools`).

### Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `content` | string | **yes** | The complete markdown document text |
| `title` | string | **yes** | Document title shown in the PDF header (e.g. `"RCA: Payment Service Timeout"`) |
| `filename` | string | no | Output filename without `.pdf` (e.g. `"rca-payment-2026-03-08"`). Defaults to a slugified version of `title`. |
| `metadata` | object | no | Key-value pairs embedded in the PDF header: `service`, `severity`, `environment`, `author`, etc. |

### Returns (JSON)

```json
{
  "doc_id":       "uuid-string",
  "download_url": "/documents/{doc_id}/download",
  "filename":     "rca-payment-service-2026-03-08.pdf",
  "size_bytes":   45312
}
```

### Example Call

```json
{
  "content":  "# RCA: Payment Service Timeout\n\n## Executive Summary\n...",
  "title":    "RCA: Payment Service Timeout",
  "filename": "rca-payment-service-2026-03-08",
  "metadata": {
    "service":     "payment-api",
    "severity":    "HIGH",
    "environment": "production"
  }
}
```

## PDF Features

- **Professional header**: NexusCode branding, incident title, coloured severity badge
- **Auto table of contents**: Generated from `##` and `###` headers
- **Code blocks**: Monospace font, grey background, border-left accent
- **Tables**: Striped rows, full-width
- **Footer**: Page numbers + "Generated by NexusCode on {datetime}"
- **Print-ready**: `@page { margin: 2cm }`, widows/orphans control

## Downloading the PDF

The `download_url` returned by the tool is a direct link to stream the PDF:

```
GET /documents/{doc_id}/download
→ Content-Disposition: attachment; filename="rca-payment-service-2026-03-08.pdf"
→ Content-Type: application/pdf
```

Always include `download_url` in your final answer so the user can access the file.

## Run History UI

After a workflow step that generated a PDF completes, the Run History page shows a
**"📥 Download PDF — filename.pdf (N KB)"** button directly on the step card.
The button fetches and serves the PDF from the server when clicked.

## Implementation Notes

- PDFs are stored as BYTEA in `generated_documents` (PostgreSQL). S3 migration path: replace `store_document` with an S3 PUT and store only the object key.
- Generation is synchronous in a thread pool executor so it does not block the async event loop.
- The `run_id` and `step_id` from the workflow executor are automatically injected via `extra_context` — the agent does not need to supply them.
